# Archive Keeper 使用说明

## 这是什么

Archive Keeper 是一个本地归档维护工具。它帮你在同一块硬盘内保留多份副本，定期检查文件完整性，发现损坏后尽可能自动修复。

适合场景：重要但不常变动的文件（合同、照片、财务文档等），数据量在 10GB 以内，每隔几个月手动维护一次。

## 前置条件

- Windows 系统（已安装 Python 3.8+）
- 只需要一个文件：`archive_keeper.py`

验证 Python 是否可用：
```
python --version
```

## 快速上手

### 第一步：初始化

选一个文件夹作为归档根目录（例如 `D:\我的归档`），运行：

```
python archive_keeper.py init D:\我的归档
```

这会创建以下结构：
```
D:\我的归档\
├── master\          ← 把你的文件放这里
├── mirrors\
│   └── mirror_1\    ← 系统自动维护的镜像
├── snapshots\       ← 历史快照
└── .archive\        ← 系统文件（配置、日志、基准）
```

### 第二步：放入文件

把你要归档的文件放进 `master` 文件夹。支持子文件夹。

```
D:\我的归档\master\
├── 合同\
│   ├── 2024年采购合同.pdf
│   └── 2025年销售合同.pdf
├── 财务\
│   └── 年度报表.xlsx
└── 照片\
    └── 公司活动.jpg
```

### 第三步：运行维护

```
python archive_keeper.py maintain D:\我的归档
```

这一条命令会依次完成：
1. 创建快照（保存当前状态）
2. 同步到镜像
3. 完整性校验
4. 必要时自动修复

最后输出结果：**通过** / **警告** / **需要处理**。

### 日常使用

每隔几个月运行一次 `maintain` 即可。如果期间修改了 master 中的文件，maintain 会自动处理同步。

## 所有命令

| 命令 | 作用 | 会修改文件吗 |
|------|------|:---:|
| `init` | 初始化归档目录 | 是 |
| `maintain` | 统一维护（推荐日常使用） | 是 |
| `sync` | 只同步主副本到镜像 | 是 |
| `verify` | 只做校验，不修改任何东西 | 否 |
| `snapshot` | 只创建快照 | 是 |
| `repair` | 只做修复 | 是 |
| `status` | 查看当前状态 | 否 |

## 预演模式

任何会修改文件的命令都支持 `--dry-run`，只检查不实际操作：

```
python archive_keeper.py maintain D:\我的归档 --dry-run
```

**建议**：不确定时先加 `--dry-run` 看看会发生什么。

## 结果含义

- **通过**：一切正常
- **警告**：有需要注意的情况（如镜像中有多余文件），但不紧急
- **需要处理**：发现损坏或无法自动判定的冲突，需要你介入

## 删除保护

如果你从 master 中删除了文件，镜像中对应的文件**不会**被自动删除。这是为了防止误删。系统会提示你手动确认。

## 日志和报告

每次运行都会在 `.archive` 目录下生成：
- `logs/` — 详细日志（排查问题用）
- `reports/` — 摘要报告（看结果用）

## 常见问题

**Q: 我能直接修改 mirrors 或 snapshots 里的文件吗？**
不要。只修改 master。mirrors 和 snapshots 由系统维护。

**Q: 快照会占很多空间吗？**
默认最多保留 10 个快照。超出后自动清理最旧的。可在 `.archive/config.json` 中修改 `max_snapshots`。

**Q: 我想用两个镜像而不是一个？**
修改 `.archive/config.json` 中的 `mirror_count` 为 2，下次 sync 会自动创建 `mirror_2`。

**Q: 系统怎么判断哪个版本是对的？**
通过"投票"：主副本、镜像、快照、校验基准，多数一致的版本被认为是正确的。如果无法判定（比如两个版本各执一词），系统会停下来让你决定。
